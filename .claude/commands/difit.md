# difit コードレビュー

difitのローカルレビューUIを起動して、コードレビューを行います。

## 1. レビュー対象の決定

`$ARGUMENTS` が指定されていればそれをそのまま使用します。
指定がない場合は `git merge-base HEAD $(git rev-parse --abbrev-ref --symbolic-full-name @{u} 2>/dev/null || echo main)` でデフォルトブランチとの差分を対象にします。

以下を並列で確認:
- `git status --short` で現在の状態を確認
- `git branch --show-current` で現在のブランチを確認

## 2. difitサーバーの起動

以下のコマンドをバックグラウンドで実行します（pseudo-TTY が必要なため `script` でラップ）:

```bash
script -q /dev/null npx --yes difit --include-untracked <target>
```

`<target>` は手順1で決定した対象を使用します。指定なしの場合は引数なしで起動します。

サーバーが起動したら、ユーザーに以下を伝えてください:

---

ブラウザで difit のレビュー画面が開きました。

**レビュー手順:**
1. 各ファイルの差分を確認する
2. コメントしたい行をクリックしてコメントを追加する
3. レビューが完了したら画面上部の「Copy All」ボタンでコメントをコピーする
4. コピーしたコメントをここに貼り付けてください

レビューコメントを貼り付けると、内容に基づいてコードを修正します。

---

## 3. レビューコメントへの対応

ユーザーがコメントを貼り付けたら、以下の手順で対応します:

1. 各コメントを読み、対象ファイルと行番号を特定する
2. 必要なファイルを Read で確認する
3. 各コメントに対して以下のいずれかを実施:
   - **修正が必要**: Edit で変更を加える
   - **対応不要**: その理由を明確に説明する
4. すべての対応が完了したら、対応内容の一覧をまとめて報告する

### 対応報告のフォーマット

```
## レビュー対応完了

### 対応した指摘
- [ファイル名:行番号] 指摘内容 → 対応内容

### 対応しなかった指摘（理由付き）
- [ファイル名:行番号] 指摘内容 → 理由
```
